# Troubleshooting

Короткий список самых частых проблем во время мастер-класса и способов их решить.

## 1. Не видно терминала

Откройте терминал через верхнее меню:

`Terminal → New Terminal`

Если меню не видно, откройте палитру команд:
- `Cmd + Shift + P` на Mac
- `Ctrl + Shift + P` на Windows

Затем найдите:

`Terminal: Create New Terminal`

---

## 2. Открылся не codespace, а просто веб-редактор GitHub

Проверьте, что вы открыли именно **codespace**, а не обычный редактор.

Правильный путь:
1. Откройте страницу `https://github.com/codespaces`
2. Найдите свой codespace
3. Откройте его в браузере

Если терминала нет, скорее всего, вы не в codespace.

---

## 3. Команда `codex` не запускается

Попробуйте проверить версию:

```bash
codex --version
```

Если команда не найдена или появляется ошибка, откройте новый codespace из шаблона репозитория.

Если проблема не исчезла, попросите ведущего помочь.

## 4. Обычный вход через браузер не работает

Иногда вход через обычный браузерный callback в cloud-среде работает нестабильно.

Используйте более надежный вариант:

```bash
codex login --device-auth
```
Дальше:

1. Откройте ссылку, которую покажет терминал
2. Войдите в ChatGPT
3. Введите одноразовый код
4. Вернитесь в терминал

## 5. После входа ничего не происходит

Подождите несколько секунд.

Если терминал завис, откройте новый терминал и снова выполните:

```bash
codex login --device-auth
```

## 6. Codespace долго грузится

Иногда GitHub Codespaces запускается не сразу.

Что можно сделать:

- подождать 1–2 минуты
- обновить страницу
- открыть codespace заново через https://github.com/codespaces

Если совсем не помогает, закройте старый codespace и создайте новый.

## 7. Появилось сообщение про слишком много running codespaces

Это значит, что у вас уже запущен другой codespace.

Что делать:

- Откройте https://github.com/codespaces
- Найдите старый codespace
- Остановите или удалите его
- После этого откройте нужный codespace снова


## 8. GitHub или codespace не открывается в браузере

Лучше использовать:

- Chrome
- Edge

Если страница не открывается:

- попробуйте другой браузер
- отключите VPN или прокси, если они мешают
- перезагрузите страницу


## 9. Codex отвечает странно или делает слишком смелые выводы

Это нормально: агент помогает искать идеи, но не заменяет проверку.

Что делать:

- попросите его показать, как именно он проверил вывод
- попросите его написать код или логику расчета
- перепроверьте найденную “аномалию” вручную

Хорошая формулировка:

_Покажи, как именно ты пришел к этому выводу, и отдельно укажи, что нужно перепроверить вручную._

## 10. График построился, но выглядит странно

Попросите Codex:

- объяснить, почему он выбрал именно такой график
- предложить более понятную альтернативу
- переподписать оси и заголовок

Например:

_Сделай этот график более понятным для читателя. Проверь подписи осей, заголовок и выбери более удачный тип визуализации, если нужно._

## 11. Что делать, если совсем застряли

Если что-то не получается:

- Не пытайтесь чинить всё сразу
- Откройте новый терминал
- Проверьте, что вы в codespace
- Попробуйте снова: **codex** или **codex login --device-auth**

Если не помогло, лучше позвать ведущего, чем тратить 10 минут на случайную проблему.

## Главное правило

Codex — это помощник для разведки в данных.

Он хорошо умеет:

- быстро читать файлы
- искать закономерности
- строить черновые графики
- предлагать идеи

**Но любой результат нужно перепроверять журналистски:**

- по данным
- по методологии
- по контексту
- по здравому смыслу
